# Data I/O

---

## Positional Args

`name` and `data` can be used positionally without keywords:

```python
u.rjb("config")              # name="config"
u.wjb({"k": "v"}, "out")    # data=, name=
u.hjb("config")
```

---

## Key Parameters

```python
data=           # data to write (required for w/u actions)
name=           # filename (extension auto-added, omit=dumps mode)
keydata=        # nested path — "user/email", "items[0]/name"
walk=True       # recursive search into subdirectories
default=        # default value when keydata is missing
append=True     # append to existing file
append_mode=    # "jsonl" | "array" | "object" | (omit=auto-detect)
jsonl=True      # read as JSONL format
dir_path=       # sub path relative to loc
```

---

## dir_path — Sub Path

Second positional argument — specifies a sub path relative to loc:

```python
# saves to data/json/base/mytest/abc/test.json
u.wjb({"k": "v"}, "test", "mytest/abc")
u.rjb("test", "mytest/abc")

# full-chain works the same
use.read.json.base("test", "mytest/abc")

# leading / means absolute path
u.rjb("test", "/external/path")
```

---

## keydata — Nested Path Access

```python
u.rjb("config", keydata="user/email")
u.rjb("data",   keydata="items[0]/name")
u.rjb("cfg",    keydata="db/host", default="localhost")
```

---

## JSON Append Patterns

```python
# JSONL (one JSON object per line)
u.wjb({"date": "2026-01-01", "user": "Alice"}, "log", append=True, append_mode="jsonl")
u.rjb("log", jsonl=True)   # → [{"date":...}, ...]

# append item to array
u.wjb([{"id": 1}], "items")
u.wjb({"id": 2}, "items", append=True, append_mode="array")

# merge into object
u.wjb({"a": 1}, "cfg")
u.wjb({"b": 2}, "cfg", append=True, append_mode="object")

# auto-detect (infers array/object from existing file structure)
u.wjb(new_item, "data", append=True)
```

## Mem Store — In-Process Memory

Stored in an in-process dict — no files. Lost on process restart.  
Format prefix is routing convention only — **all formats share the same `_STORE`**.

```python
# Basic CRUD
u.wjm(data, "key")          # write (0.005ms)
u.rjm("key")                # read
u.rjm("key", default=None)  # default when missing
u.ujm({"b": 99}, "key")    # dict merge (update if dict, overwrite otherwise)
u.djm("key")                # delete
u.hjm("key")                # check existence → bool
u.ljm()                     # all keys → list

# other format prefixes work identically (same _STORE)
u.wym(data, "key")  / u.rym("key")   # yaml
u.wtm(text, "key")  / u.rtm("key")   # txt
u.wmm(md, "key")    / u.rmm("key")   # md
u.wcm(rows, "key")  / u.rcm("key")   # csv
u.wam(data, "key")  / u.ram("key")   # any
```

**Execution Cache Pattern:**
```python
# cache expensive result in memory
u.wjm(expensive_result, "cache")

# reuse later — no file I/O, no context re-exposure
data = u.rjm("cache")
```

**ejm — emit (serialization only, separate from mem)**
```python
u.ejm({"k": "v"})            # → str (no file, no storage)
u.ejm({"k": "v"}, type="d")  # str/bytes → dict deserialization
u.ejm(data, type="b")        # → bytes
u.ejm(data, type="l")        # → list (line by line)
```

---

## Constraints

- `u.ejm()` — serialize only. No storage, returns `str`.
- mem store — lost on process restart. Use `wjb`/`wjc` for persistence.
- `u.djb(name="*")` — wildcard delete is blocked.
- `name` not specified → dumps mode (no file created).
